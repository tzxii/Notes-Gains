3.完整的使用git的流程（每到一台新的机器，新的操作系统里面）：
（1）git --version确保git已经安装 。  记得本地也需要先有git仓库才行，也就是git init
（2）git config --global user.name "你的名字"
		git config --global user.email "你的GitHub邮箱"
	设置对应的提交者信息。
（3）ls ~/.ssh查看是否已经有ssh密钥，没有则生成：
	ssh-keygen -t ed25519 -C "3140749482@qq.com"
	cat ~/.ssh/id_ed25519.pub 把这个复制到github里的setting
	ssh -T git@github.com 查看是否能连接到github
（4）git remote -v确认一下远程仓库
	git remote set-url xxxxxx
（5）add commit push


4.git pull --rebase origin main 调整自己本地的，适配远端的仓库。也就是减少分支，而是把自己的此次修改直接和远端仓库拼成一个分支。
记住这条“工程判断准则”：

只有你一个人用的分支
👉 rebase（整理自己）

已经给别人用 / 公共分支
👉 merge（尊重历史）
